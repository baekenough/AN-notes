## Why metric discipline matters

Without metric discipline, resilience work becomes anecdotal.
Teams need measurable signals to decide where to invest effort.

## Core metric model

| Metric | Decision use | Data source |
|---|---|---|
| MTTD | alert quality and routing | monitoring timeline |
| MTTC | mitigation decision speed | incident decision log |
| MTTR | service restoration efficiency | deploy + verification output |
| Evidence freshness | closure confidence quality | command evidence archive |
| Follow-up burn-down | hardening execution health | action backlog |

## Severity-specific SLO targets

- **SEV-1**: containment + rollback path activation in strict response window
- **SEV-2**: major degradation stabilized in agreed team window
- **SEV-3**: planned corrective release completed in standard cycle

Use numeric targets and review variance weekly.

## Data quality protocol

Capture five mandatory timestamps:

- incident start
- first detection
- first mitigation decision
- stable-state verification
- closure

Timestamp gaps should be treated as process defects.

## Weekly resilience review cadence

- inspect breaches vs target
- analyze repeated tail-latency incidents
- map misses to control backlog categories
- assign owner + due date for top regressions

## Threshold-to-escalation policy

Per metric define:

- green: normal
- yellow: watch
- red: immediate escalation

Red state must auto-create a named owner path.

## Dashboard rules

- display trend and percentile, not just average
- separate severity lanes
- show context denominator
- link every spike to incident records

## Quarterly SLO calibration

Every quarter:

- adjust SLOs only after sustained attainment
- remove metrics unused in decisions
- add one metric for newly emerging failure class

## Advanced anti-patterns

### High metric count, low action rate

If metrics do not drive actions, they are noise.

### SLO ownership not assigned

Unowned SLOs degrade silently.

### Closure with stale verification evidence

Stale evidence cannot justify current-state confidence.

## Quick checklist

Before monthly reliability council:

- metric definitions current
- severity SLOs published
- red/yellow breaches owned
- follow-up burn-down reviewed

Gemini CLI accelerates execution. Metrics keep reliability accountable.
