## 왜 headless mode가 중요한가?

대화형 채팅은 Gemini CLI의 절반만 보여 줍니다. 나머지 절반은 **자동화**입니다. 셸 스크립트, CI 작업, 콘텐츠 파이프라인, 에디터 보조 도구 안에서 Gemini CLI를 돌릴 수 있다는 뜻입니다.

공식 문서는 headless mode를 **터미널 UI 없이 텍스트나 JSON을 반환하는 비대화형 실행 방식**으로 설명합니다.

## 언제 headless mode로 들어가나?

공식 문서 기준으로 headless mode는 아래 경우에 시작됩니다.

- **non-TTY** 환경에서 실행될 때
- `-p` / `--prompt`로 직접 프롬프트를 줄 때

예시:

```bash
gemini -p "이 저장소의 아키텍처를 요약해줘"
```

## 구조화된 출력 포맷

### 일반 텍스트

```bash
gemini -p "이 저장소를 요약해줘"
```

### JSON 출력

```bash
gemini -p "이 저장소를 요약해줘" --output-format json
```

문서 기준 JSON 객체에는 보통 다음이 포함됩니다.

- `response`
- `stats`
- optional `error`

### 스트리밍 JSONL 출력

```bash
gemini -p "이 파일들을 리뷰해줘" --output-format jsonl
```

유용한 이벤트 타입:

- `init`
- `message`
- `tool_use`
- `tool_result`
- `error`
- `result`

즉 단순 채팅 로그가 아니라, 커스텀 도구와 연결하기 쉬운 형태입니다.

## 종료 코드는 자동화에서 중요하다

공식 headless 문서가 제공하는 종료 코드는 아래와 같습니다.

| 종료 코드 | 의미 |
|-----------|------|
| `0` | 성공 |
| `1` | 일반 오류 또는 API 실패 |
| `42` | 잘못된 입력/인자 |
| `53` | turn limit 초과 |

CI에서 문장 파싱보다 훨씬 믿을 만한 신호입니다.

## 자동화에서는 인증 방식도 바뀐다

인증 문서는 headless mode에서 보통 아래 둘 중 하나를 쓰라고 안내합니다.

- **Gemini API key**
- **Vertex AI**

브라우저 로그인이 필요한 Google 계정보다 자동화에는 훨씬 잘 맞습니다.

예시:

```bash
export GEMINI_API_KEY="YOUR_GEMINI_API_KEY"
gemini -p "@content 빠진 Gemini CLI 주제를 나열해줘" --output-format json
```

## 실전 자동화 패턴

### 1) 콘텐츠 QA 스크립트

```bash
gemini -p "@content ko, en, es의 gemini-cli 글을 비교해 빠진 slug나 불일치한 주장을 찾아줘" --output-format json
```

### 2) 다른 도구로 응답 넘기기

```bash
gemini -p "@src에서 TODO 후보를 요약해줘" --output-format json   | jq -r '.response'
```

### 3) 머신 친화적인 이벤트 스트림 받기

```bash
gemini -p "@content/en/gemini-cli를 리뷰해줘" --output-format jsonl
```

진행 상황 표시, tool trace 저장, 후속 자동화 연결에 좋습니다.

## 비용/속도 관점 메모

API key 또는 Vertex AI를 쓰면 Gemini CLI가 **token caching**을 활용할 수 있습니다. 반복 자동화일수록 동일 컨텍스트 재사용 비용을 줄이는 데 도움이 됩니다.

표준화 전에 interactive 세션에서 `/stats`를 확인해 두면 좋습니다.

## 언제 Gemini 자동화가 더 맞나?

| 상황 | 적합한 도구 | 이유 |
|------|-------------|------|
| JSON 출력이 필요한 저장소 분석 | Gemini CLI | headless + 도구 사용 가능 |
| 강한 격리가 필요한 코드 실행 | GPT Codex | 샌드박스 중심 |
| 사람과 길게 주고받는 편집 | Claude Code | 대화형 루프 강점 |

## 이 저장소에서의 좋은 사용처

AN-Con은 이미 다국어 MDX 콘텐츠를 구조적으로 가지고 있습니다. 그래서 Gemini CLI 자동화는 아래 작업에 특히 잘 맞습니다.

- 로케일 parity 점검
- 콘텐츠 공백 탐지
- frontmatter 감사
- 구조화된 JSON 형태의 rewrite 제안 생성

즉, 먼저 interactive로 워크플로를 찾고, 그다음 유용한 부분만 headless 스크립트로 굳히면 됩니다.
