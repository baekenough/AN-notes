> **공식 레퍼런스**: [Headless mode](https://geminicli.com/docs/cli/headless/) · [Authentication](https://geminicli.com/docs/get-started/authentication/)

## 왜 headless mode가 중요한가?

대화형 채팅은 Gemini CLI의 절반만 보여 줍니다. 나머지 절반은 **자동화**입니다. 셸 스크립트, CI 작업, 콘텐츠 파이프라인, 에디터 보조 도구 안에서 Gemini CLI를 돌릴 수 있다는 뜻입니다.
공식 문서는 headless mode를 **터미널 UI 없이 텍스트나 JSON을 반환하는 비대화형 실행 방식**으로 설명합니다.

## 언제 headless mode로 들어가나?

- **non-TTY** 환경(CI/CD 파이프라인 등)에서 실행될 때
- `-p` / `--prompt`로 직접 프롬프트를 줄 때

예시:
```bash
gemini -p "이 저장소의 아키텍처를 요약해줘" --output-format json
```

`--output-format json` (또는 `jsonl`)을 사용하면 응답 결과를 파싱하기 쉬운 객체(`response`, `stats`, `error`)로 반환합니다.

## 자동화에서는 인증 방식도 바뀐다

인증 문서는 headless mode에서 브라우저 기반 로그인이 아닌 다음 방식을 권장합니다.
- **Gemini API key** (`GEMINI_API_KEY` 환경변수)
- **Vertex AI** 서비스 어카운트

이 방식을 쓰면 **토큰 캐싱(Token Caching)**도 자동으로 활용되어 반복되는 CI 파이프라인의 비용을 크게 절감할 수 있습니다.

## 실전 자동화 패턴

### 1) 콘텐츠 QA 스크립트
```bash
gemini -p "@content ko, en, es의 gemini-cli 글을 비교해 빠진 slug나 불일치한 주장을 찾아줘" --output-format json
```

### 2) 파이프라인 연결 (jq 활용)
```bash
gemini -p "@src에서 보안 취약점 요약해줘" --output-format json | jq -r '.response'
```

### 3) 머신 친화적인 이벤트 스트림 (`jsonl`)
```bash
gemini -p "@content/en/gemini-cli를 리뷰해줘" --output-format jsonl
```
진행 상황 표시(`tool_use`, `tool_result` 이벤트), tool trace 저장, 후속 자동화 연결에 좋습니다.

### 4) 실전 예제: GitHub Actions 연동 (자동 코드 리뷰)

PR이 올라올 때마다 Gemini CLI가 코드를 스캔하고 린트 및 패턴 검사를 수행하도록 만드는 전형적인 워크플로 예시입니다.

```yaml
name: Gemini CLI Code Review
on: [pull_request]

jobs:
  gemini-review:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout Repository
        uses: actions/checkout@v4
        with:
          fetch-depth: 0 # 전체 히스토리를 가져와 diff 파악에 유리

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install Gemini CLI
        run: npm install -g @google/gemini-cli

      - name: Run Headless Review
        env:
          GEMINI_API_KEY: ${{ secrets.GEMINI_API_KEY }}
        run: |
          gemini -p "최근 커밋된 파일들을 읽고 팀의 GEMINI.md 컨벤션을 지키고 있는지 검토한 후 JSON으로 리포팅해줘" --output-format json > review.json
          cat review.json
```

## 이 저장소에서의 좋은 사용처

AN-Con은 이미 다국어 MDX 콘텐츠를 구조적으로 가지고 있습니다.
대화형 모드로 먼저 로케일 패리티 점검이나 frontmatter 검증 워크플로를 발견한 뒤, 그 유용한 프롬프트를 GitHub Actions의 headless 스크립트로 굳히는 방식을 추천합니다.
