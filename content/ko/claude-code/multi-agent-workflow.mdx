## 서브에이전트란?

Claude Code는 전문화된 서브에이전트를 생성할 수 있습니다 — 집중된 작업을 병렬로 처리하는 별도의 Claude 인스턴스입니다. Claude가 팀에 작업을 위임하는 것으로 생각하세요.

## 서브에이전트 사용 시점

| 시나리오 | 단일 에이전트 | 서브에이전트 |
|---------|------------|------------|
| 오타 수정 | ✅ | 과도함 |
| 파일 3개 리뷰 | ✅ | ✅ (더 빠름) |
| 파일 20개 리팩토링 | 너무 느림 | ✅ |
| 리서치 + 구현 | ✅ | ✅ (병렬) |
| 크로스 모듈 디버깅 | 컨텍스트 손실 | ✅ |

## 오케스트레이터 패턴

메인 대화가 오케스트레이터 역할을 합니다 — 작업을 계획하고 전문가에게 위임합니다:

```
사용자: "인증 모듈의 보안 문제를 리뷰해줘"

Claude (오케스트레이터):
  → 에이전트 1 생성: auth/login.ts의 인젝션 위험 리뷰
  → 에이전트 2 생성: auth/session.ts의 토큰 처리 리뷰
  → 에이전트 3 생성: auth/middleware.ts의 우회 위험 리뷰
  → 결과를 단일 보고서로 집계
```

## 실용 예시: 대규모 리팩토링

```
모든 API 라우트를 Express 미들웨어 패턴에서 Next.js Route Handler로 마이그레이션해줘.

src/api/에 15개 라우트가 있어. 병렬 에이전트 사용해 —
라우트 그룹별로 하나씩 (auth, users, products, orders, analytics).

src/api/utils/의 공유 유틸리티 함수는 수정하지 마.
```

Claude가 수행할 작업:
1. 라우트 그룹 분석
2. 5개 병렬 에이전트 생성
3. 각 에이전트가 3개 라우트 처리
4. 결과 집계 및 충돌 해결

## Agent Teams (실험적)

Claude Code의 Agent Teams 기능은 에이전트 간 P2P 통신을 가능하게 합니다. 모든 것이 오케스트레이터를 통하지 않고:

```
리뷰어 에이전트 ←→ 구현자 에이전트
     ↕                    ↕
  테스터 에이전트 ←→ 문서화 에이전트
```

다음에 적합:
- 코드 리뷰 → 수정 → 재리뷰 사이클
- 크로스 모듈 복잡한 디버깅
- 토론과 종합이 필요한 리서치

## 백그라운드 에이전트

에이전트를 백그라운드로 실행하면 기다리지 않고 다른 작업을 계속할 수 있어요. 시간이 오래 걸리는 작업에 특히 유용해요.

```
대규모 리팩토링을 백그라운드로 실행해줘.
완료되면 알려줘. 그동안 나는 다른 작업을 할게.
```

백그라운드 실행이 적합한 경우:
- **대규모 리팩토링** — 수십 개 파일을 변환하는 작업
- **긴 테스트 실행** — 전체 테스트 스위트 실행
- **코드 리뷰** — 대형 PR의 전체 파일 분석
- **의존성 분석** — 프로젝트 전체 의존성 트리 탐색

에이전트가 완료되면 결과와 함께 알림을 받을 수 있어요. 다른 작업을 진행하다가 결과를 확인하면 됩니다.

## Git Worktree를 활용한 격리

각 에이전트가 독립적인 git worktree에서 작업하면 메인 브랜치에 영향 없이 실험할 수 있어요.

```
각 에이전트가 별도 worktree에서 작업하도록 해줘:
- 에이전트 1: feature/auth-refactor 브랜치에서 인증 모듈 리팩토링
- 에이전트 2: feature/api-migration 브랜치에서 API 마이그레이션
- 에이전트 3: feature/test-coverage 브랜치에서 테스트 보강

각각 독립적으로 작업하고 결과를 별도 브랜치로 제출해줘.
```

Worktree 격리의 장점:
- **안전한 실험** — 실패해도 worktree만 삭제하면 끝
- **병렬 브랜치 작업** — 여러 기능을 동시에 개발
- **충돌 없는 협업** — 에이전트들이 서로 간섭하지 않음
- **쉬운 롤백** — 마음에 들지 않으면 브랜치 삭제

## 에이전트 전문화 패턴

에이전트에게 특정 전문 역할을 부여하면 훨씬 깊이 있는 분석이 가능해요.

```
프로젝트 전체 분석을 해줘. 다음 전문 에이전트를 사용해:

보안 전문 에이전트:
  - OWASP Top 10 기준으로 취약점 점검
  - SQL 인젝션, XSS, CSRF 위험 탐지
  - 인증/인가 로직 검토

성능 전문 에이전트:
  - N+1 쿼리 패턴 탐지
  - 메모리 누수 가능성 분석
  - 불필요한 재렌더링 및 무거운 연산 탐지

테스트 전문 에이전트:
  - 현재 코드 커버리지 측정
  - 누락된 엣지 케이스 식별
  - 통합 테스트가 부족한 모듈 목록화
```

각 에이전트가 자신의 전문 영역에 집중하기 때문에 단일 에이전트보다 훨씬 상세한 결과를 얻을 수 있어요.

## 실전 시나리오: 프론트엔드 + 백엔드 동시 작업

새 기능을 추가할 때 프론트엔드, 백엔드, DB를 동시에 작업할 수 있어요:

```
"사용자 프로필" 기능을 추가해줘. 세 에이전트가 동시에 작업해:

에이전트 1 (백엔드): GET/PUT /api/users/:id 엔드포인트 구현
에이전트 2 (프론트엔드): ProfilePage, ProfileForm React 컴포넌트 구현
에이전트 3 (DB): users 테이블에 bio, avatar_url 컬럼 추가 마이그레이션

공유 타입 정의는 src/types/user.ts를 참고해.
완료 후 각 결과를 조합해서 통합 계획을 제안해줘.
```

세 에이전트가 동시에 작업하기 때문에 순차 작업 대비 시간을 크게 줄일 수 있어요.

## 에이전트 사용 시 주의사항

에이전트가 항상 더 좋은 건 아니에요. 몇 가지를 고려해야 해요:

- **단순 작업은 오히려 느려져요** — 오타 수정, 파일 하나 편집 같은 작업은 단일 에이전트가 빨라요. 에이전트 생성 오버헤드가 있거든요.
- **컨텍스트가 분리돼요** — 각 에이전트는 별도 컨텍스트를 가져요. 에이전트 간 공유가 필요하면 파일이나 태스크 목록을 통해야 해요.
- **비용이 늘어나요** — 에이전트 수 × API 호출 = 비용 증가. 꼭 필요할 때만 사용하는 게 좋아요.
- **4개 이상은 효율이 떨어질 수 있어요** — Claude Code는 최대 4개 동시 실행을 권장해요. 그 이상은 관리 오버헤드가 커져요.

## Codex 비교

GPT Codex는 각 작업을 단일 샌드박스 에이전트로 실행합니다. 멀티 에이전트 오케스트레이션은 없지만, 샌드박스 모델이 태스크별 전체 컨테이너 격리를 제공합니다.

| 기능 | Claude Code 서브에이전트 | Codex 태스크 |
|-----|----------------------|------------|
| 병렬성 | 최대 4개 동시 | 동시 태스크 가능 |
| 통신 | 공유 컨텍스트, 메시징 | 독립적, 격리됨 |
| 상태 공유 | 파일과 태스크 목록 | 커밋된 코드 |
| 적합한 작업 | 복잡하게 연결된 작업 | 독립적, 잘 정의된 작업 |
