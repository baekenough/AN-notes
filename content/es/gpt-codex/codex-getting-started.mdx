## Empieza por un bucle exitoso, no por todas las funciones

La forma más rápida de sentirte cómodo con Codex no es memorizar todas las settings.
Es completar una tarea pequeña y segura de punta a punta.

Al principio solo necesitas responder cuatro preguntas:

- ¿Qué cliente vas a usar?
- ¿Qué repo vas a abrir?
- ¿Qué tan pequeña será la primera tarea?
- ¿Cómo vas a revertir si no te gusta el resultado?

## ¿Dónde conviene empezar?

Codex puede usarse desde la app, el CLI, la extensión de IDE y flujos cloud.

### App de Codex
Suele ser la mejor puerta de entrada.
Review, automations y worktrees están visibles en un solo lugar.

### CLI
Ideal si trabajas desde la terminal.
Te deja ver con mucha claridad prompts, comandos, diffs y checks.

### Extensión de IDE
Útil si prefieres quedarte dentro del editor.
Aun así, en las primeras sesiones la app o el CLI suelen explicar mejor el comportamiento real.

### Cloud
No hace falta empezar aquí.
Primero consigue un flujo local que ya entiendas; después súbelo a cloud.

## Tus primeros 10 minutos

El flujo del Quickstart es simple:

1. Instalar la app de Codex o el CLI
2. **Iniciar sesión con tu cuenta de ChatGPT o una OpenAI API key**
3. Elegir la carpeta del proyecto
4. Enviar la primera tarea

La documentación aclara que con API key algunas funciones pueden estar limitadas. Para la mayoría de la gente que recién empieza, la combinación más simple es **proyecto local + inicio de sesión normal**.

## Empieza local antes de empezar ambicioso

Si intentas aprender cloud, automations, permisos amplios y diseño de prompts al mismo tiempo, se vuelve difícil saber qué falló.

Mejores primeras tareas:

- “Explícame este repo en cinco bullets.”
- “Encuentra los archivos más relevantes para este error.”
- “Identifica candidatos de dead code en esta carpeta.”
- “Haz un refactor pequeño sin cambiar comportamiento.”

En otras palabras: trabajo fácil de inspeccionar y fácil de revertir.

## Usa checkpoints de Git temprano

Quickstart recomienda checkpoints, y con razón: Codex puede cambiar bastante en poco tiempo.

Un bucle sencillo para empezar:

```bash
git status
git add -A && git commit -m "checkpoint: before codex task"
```

Si el resultado te gusta, crea otro checkpoint después.
Eso reduce mucho la sensación de “creo que mejoró, pero no sé bien qué pasó”.

## Cuatro comandos para aprender primero

La guía de slash commands tiene muchos comandos, pero al principio casi siempre bastan estos:

### `/init`
Genera un scaffold de `AGENTS.md` en el directorio actual.
Apréndelo en cuanto te canses de repetir reglas del repo.

### `/review`
Pide a Codex una revisión de tu working tree.
Es el segundo pase más útil después de implementar.

### `/mcp`
Muestra qué herramientas MCP están disponibles en la sesión.
Útil cuando necesitas docs externas, trackers o sistemas internos.

### `/status`
Te enseña modelo activo, approval policy, writable roots y estado de la sesión.
Si Codex se comporta raro, empieza aquí.

## Tu primer prompt debe sonar a trabajo, no a deseo

Evita “mejora toda la app”.
Mejor algo como esto:

```text
Goal:
Encuentra utilidades sin uso dentro de src/lib/.

Constraints:
- Solo investigar; no modificar archivos
- Mostrar evidencia de import o uso para cada candidato

Done when:
- Listar los tres candidatos principales con sus razones
```

Eso le da a Codex un objetivo y una línea de llegada.

## Checklist de la primera semana

En la primera semana suele funcionar bien este orden:

1. Tareas pequeñas y locales de solo lectura
2. Cambios pequeños con lint/test
3. Revisar el diff otra vez con `/review`
4. Crear un borrador de `AGENTS.md` con `/init`
5. Subir trabajo repetido a skills o automations solo cuando el flujo ya sea estable

El orden sano es **tarea → review → reglas**, no “encender todo”.

## Resumen rápido

- Empieza con la **app o el CLI sobre un repo local**.
- La primera tarea debe ser pequeña y reversible.
- Usa checkpoints de Git desde el día uno.
- Aprende pronto `/init`, `/review`, `/mcp` y `/status`.
- Profundiza en instrucciones, sandboxing y task design cuando ese primer loop ya te resulte natural.
