## Por qué importa el modo headless

El chat interactivo es solo la mitad de Gemini CLI. La automatización lo lleva a pipelines CI/CD, scripts de bash y tooling.
La documentación define el modo headless como una ejecución que devuelve JSON o texto estructurado sin terminal interactiva.

## Cómo se activa

Se activa cuando:
- Corre en un entorno **non-TTY** (como GitHub Actions).
- Se pasa un prompt directo con `-p` / `--prompt`.

Ejemplo:
```bash
gemini -p "Resume la arquitectura de este repositorio" --output-format json
```
Con `--output-format json`, obtienes un objeto parseable con `response`, `stats` y `error`.

## La autenticación correcta ahorra costos

En automatización, la documentación aconseja evitar el login de navegador y usar:
- **Gemini API key** (`GEMINI_API_KEY`)
- **Vertex AI**

Usar estos métodos habilita automáticamente el **Token Caching**, haciendo que los pipelines repetitivos sean extremadamente baratos.

## Patrones prácticos de automatización

### 1) QA de contenido en script
```bash
gemini -p "@content compara los artículos entre ko, en y es y reporta slugs faltantes" --output-format json
```

### 2) Integración con jq
```bash
gemini -p "Enumera vulnerabilidades desde @src" --output-format json | jq -r '.response'
```

### 3) JSONL en streaming
```bash
gemini -p "Revisa @content/en/gemini-cli" --output-format jsonl
```
Ideal para mostrar progreso de herramientas (`tool_use`) en interfaces propias.

### 4) Ejemplo Real: GitHub Actions (Review automático)

Un pipeline clásico donde Gemini analiza los Pull Requests en busca de fallos o faltas al `GEMINI.md`.

```yaml
name: Gemini CLI Code Review
on: [pull_request]

jobs:
  gemini-review:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout Repository
        uses: actions/checkout@v4
        with:
          fetch-depth: 0 # Necesario para entender los diffs

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install Gemini CLI
        run: npm install -g @google/gemini-cli

      - name: Run Headless Review
        env:
          GEMINI_API_KEY: ${{ secrets.GEMINI_API_KEY }}
        run: |
          gemini -p "Revisa los archivos modificados y comprueba que cumplan las reglas de GEMINI.md. Devuelve un reporte JSON." --output-format json > review.json
          cat review.json
```

## Un buen uso dentro de este proyecto

Descubre un flujo útil en modo interactivo (ej: validar frontmatter de MDX), y luego empaquétalo usando el código superior en tus GitHub Actions.
