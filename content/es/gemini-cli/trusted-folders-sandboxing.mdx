## Gemini CLI tiene un modelo de seguridad bastante claro

Si temes que la IA arruine tu código, comprende sus múltiples líneas de defensa:

1. **Trusted Folders**: Controla qué workspaces desbloquean capacidades completas.
2. **Plan Mode**: Entorno de solo lectura para diseñar antes de tocar el código.
3. **Sandboxing**: Aísla comandos destructivos de tu host.
4. **Checkpointing**: Crea puntos de retorno automáticos antes de escribir.

## Plan Mode: Diseño seguro antes de la ejecución (Read-Only)

En refactorizaciones enormes o repositorios ajenos, **Plan Mode** (`/plan` o `enter_plan_mode`) es tu mejor escudo.

- Obliga al agente a usar **únicamente herramientas de solo lectura**. No puede modificar archivos ni ejecutar shells destructivos.
- Su único propósito aquí es mapear el código y crear un documento de diseño arquitectónico. Solo cuando tú apruebas ese plan, cambias al modo normal para aplicar la ejecución.

## Trusted folders: protegerte de workspaces riesgosos

Los trusted folders vienen **desactivados por defecto**. Actívalo en tu configuración:

```json
{
  "security": {
    "folderTrust": {
      "enabled": true
    }
  }
}
```

En un repositorio *no confiable* (como un repo open-source cualquiera o código de cliente), Gemini CLI bloqueará la carga de `.env`, los MCP y la auto-aceptación de herramientas.

## Sandboxing: aislar acciones con efectos laterales

Gemini CLI puede ejecutar modificaciones y shell dentro de un contenedor en lugar de tu host.

```bash
gemini --sandbox
```

Si el proyecto requiere binarios específicos, pon un Dockerfile en `.gemini/sandbox.Dockerfile`. Es la mejor forma de mantener tu SO limpio.

## Checkpointing: crear un punto de retorno automático

Para evitar sorpresas, activa el checkpointing en `settings.json`:

```json
{
  "general": {
    "checkpointing": {
      "enabled": true
    }
  }
}
```

El agente creará un "shadow commit" en Git y guardará la conversación antes de reescribir archivos.
Si algo sale mal, simplemente:
```text
/restore
```

## Cómo lo usaría en la práctica

- **Repo desconocido**: Activa Trusted folders globalmente -> No confíes -> Usa **Plan Mode** para entender -> Usa **Sandboxing** para aislar pruebas.
- **Repo propio**: Confía en la carpeta -> Modo interactivo normal -> Activa **Checkpointing** si la edición será masiva.
