## El titular de 1M tokens es real, pero la estrategia importa más

El repositorio oficial presenta **1M tokens** como una ventaja principal. En la práctica eso sí permite razonamiento a nivel de repositorio que otras herramientas suelen aproximar.

Pero la lección real no es "mete todo el repo en cada prompt". La forma útil suele ser:
- hacer una **pasada amplia una vez**
- luego seguir con **turnos enfocados**
- mover reglas repetidas a `GEMINI.md`

## Eficiencia del contexto: Por qué ahorrar tokens importa

Aunque tengas 1 millón de tokens, leer archivos enteros sin necesidad (`read_file`) aumenta la latencia y los costos. Para usar Gemini CLI como un experto, necesitas **lecturas quirúrgicas**:

1. **Búsqueda en paralelo**: Usa `grep_search` con límites moderados (`total_max_matches`) y filtros (`include_pattern`) en paralelo.
2. **Contexto en búsquedas**: Pide líneas adyacentes (`context`, `before`, `after`) para que el agente no gaste un turno extra abriendo el archivo completo.
3. **Rangos específicos**: Usa `start_line` y `end_line` para leer solo el fragmento necesario de archivos gigantes.

## Patrones que sí funcionan bien

### 1) Mapear arquitectura antes de editar

```bash
gemini "@src @content @package.json mapea la arquitectura de esta app, su pipeline de contenido y los riesgos de mantenimiento"
```

### 2) Delegar a Sub-agentes: Codebase Investigator

Si el repositorio es inmenso, delega el análisis al sub-agente `codebase_investigator`.
En lugar de saturar el historial principal, pídele: "Encuentra la causa raíz del bug de autenticación" o "Mapea toda la estructura de rutas". El sub-agente navegará el código por su cuenta y devolverá un resumen destilado, ahorrando mucho contexto.

### 3) Analizar cambios transversales

```bash
gemini "@src @package.json Queremos cambiar el modelo de contenido. Muéstrame qué partes afectarían: loaders, rutas, UI y localización."
```

## Cómo no desperdiciar la ventana de contexto

### Mejor inclusión estructurada que prompts vagos

Malo:
```text
Lee todo y dime qué importa.
```

Mejor:
```text
@src @content @messages
Explica la arquitectura de contenido, la estrategia de locales y dónde podría romperse la paridad de slugs.
```

### El token caching cambia el costo real

La documentación oficial indica que **token caching** está disponible cuando autenticas con **Gemini API key** o **Vertex AI**. No está disponible actualmente para login de Google normal.

Los flujos grandes salen mejor con:
- `GEMINI_API_KEY`, o
- autenticación de Vertex AI

Usa `/stats` para inspeccionar consumo y ahorro por tokens cacheados.
